# About KafuOps

KafuOps exists because backend debugging is still too manual.

Observability tools show symptoms. Coding agents can write patches. But production debugging requires a bridge between runtime evidence and repository understanding.

KafuOps is that bridge.

## Philosophy

- Production evidence matters.
- Context should be selected, not dumped.
- AI-generated code must be reviewable.
- Humans stay in control.
- Memory improves future debugging.
- Privacy is a product feature, not an afterthought.

## What “Kafu” means in the product

KafuOps should feel like a reliable engineering teammate: careful, useful, and focused on helping the team fix real incidents.
