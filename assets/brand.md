# KafuOps brand

A short reference for anyone styling KafuOps surfaces — docs, slides, READMEs.

## Name

**KafuOps** — capital K, capital O, no space. Lowercase `kafuops` for CLI,
package name, and URLs.

Origin: `kafu` is borrowed from the Arabic root *kāfū / كافٍ* — "sufficient,
enough, capable". The product is an "ops layer that is sufficient" — it does
the minimum required to turn an incident into a reviewable fix, and stops.

## Palette

| Role | Hex | Notes |
|---|---|---|
| Primary (ink) | `#0F766E` | Teal 700. Trustworthy, technical, calm. |
| Accent (incident) | `#F59E0B` | Amber 500. Used for alerts, the "incident" dot in the logo, CTAs. |
| Surface (dark) | `#0B1220` | Hero backgrounds, code blocks. |
| Surface (light) | `#F8FAFC` | Light theme. |
| Muted text | `#475569` | Secondary copy. |
| Border | `#E2E8F0` | Light dividers. |

## Type

Use a clean monospace for the wordmark (JetBrains Mono, IBM Plex Mono,
Fira Code) and a humanist sans for body text (Inter, IBM Plex Sans).
Fall back to system fonts; the project does not bundle webfonts.

## Voice

- Short sentences.
- Operational, not marketing.
- Concrete numbers and file paths beat adjectives.
- When you describe what it does, also describe what it does not do.

## Logo usage

- Minimum padding equal to half the icon corner radius.
- Do not place on noisy photographic backgrounds.
- The amber dot in the lower-right of the mark is the "incident" — do not
  recolor it. The dot is the only colored detail; everything else can flip
  between teal and white.
