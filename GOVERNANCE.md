# Governance

This is a proposed governance model for KafuOps.

## Maintainers

Maintainers are responsible for:

- Reviewing contributions.
- Protecting security and privacy principles.
- Managing releases.
- Maintaining documentation quality.
- Deciding roadmap priorities.

## Decision principles

KafuOps decisions should prioritize:

1. User safety.
2. Privacy.
3. Debugging usefulness.
4. Reviewability.
5. Extensibility.
6. Simplicity.

## Project boundaries

KafuOps should remain focused on production debugging and incident-to-MR workflows. General-purpose coding-agent features should be added only when they improve that mission.
