The product rule

KafuOps will not stream all your logs to a model. It watches locally, decides when an event is a real incident, ships a small sanitized bundle to the model, and runs the resulting patch through a sandbox before any human sees a diff.

What KafuOps does

  • Observes logs, traces, runtime errors, and alert webhooks.
  • Builds a living .kafuops/memory/ map of your backend.
  • Deduplicates noisy events into incidents.
  • Selects the right source files, tests, and log snippets per incident.
  • Generates a regression test and a small patch.
  • Runs install + tests in a sandbox before opening an MR.
  • Scores confidence and blast radius, and writes a grounding manifest.

What KafuOps does not do

  • Stream continuous logs to the model.
  • Upload your whole repo by default.
  • Send .env, secrets, or private keys to the model.
  • Modify auth, payments, or migrations without approval policy.
  • Auto-merge production fixes by default.
  • Replace your observability stack.

Key features

Each feature exists in src/ today. See STATUS.md for the honest map.

Incident-triggered AI

The model is only called after a configured trigger fires. Logs and stack traces are treated as untrusted data, not instructions.

Living project memory

A scanned map of routes, services, tests, dependencies, and the architecture graph — written to disk so humans can read it.

Evidence-backed MRs

Every MR ships with root cause, files inspected, tests run, confidence score, blast radius, and a grounding manifest.

Privacy by default

Redaction at ingest, redaction before model calls, ReDoS-guarded user patterns, deny-listed paths, raw-body HMAC for webhooks.

Policy gates

Never-modify globs, require-approval globs, confidence thresholds, and post-apply checks against the actual diff.

Self-hostable

Single Docker image. Local sandbox. No vendor lock-in. AGPL-3.0.

Quick start

From a clean clone:

# install deps and build the CLI
npm install
npm run build

# inside any Node, TS, FastAPI, or Flask repo
kafuops init -y
kafuops doctor
kafuops scan

# observe a child process for crashes
kafuops run -- node ./server.js

# or drive the full pipeline manually
kafuops simulate error
kafuops incidents list
kafuops incidents open-mr inc_2026_05_15_xyz --in-place --dry-run

Without OPENAI_API_KEY and KAFUOPS_GIT_TOKEN the whole pipeline runs in dry-run, so you can inspect the generated incident, context bundle, grounding manifest, confidence score, and MR body before connecting anything live.
