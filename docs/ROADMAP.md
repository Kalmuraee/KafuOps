# Roadmap

## Phase 0: Blueprint

- Project docs.
- Architecture design.
- Setup wizard spec.
- Config schema.
- MR template.

## Phase 1: MVP

- CLI: `init`, `doctor`, `scan`, `run`.
- TypeScript/Node repository scanner.
- Local wrapper mode.
- Rolling log buffer.
- Stack trace detection.
- Incident creation.
- Project memory generation.
- OpenAI root-cause analysis.
- Context bundle and grounding manifest.
- GitHub or GitLab MR creation.

## Phase 2: Safer fixes

- Regression test generation.
- Patch sandbox.
- Confidence scoring.
- Blast-radius analysis.
- Policy engine.
- Redaction engine.
- Audit UI or report export.

## Phase 3: Production integrations

- Sidecar agent.
- OpenTelemetry ingestion.
- Sentry webhook.
- Alertmanager webhook.
- Datadog webhook.
- GitHub and GitLab support.
- Docker deployment.
- Kubernetes deployment.

## Phase 4: Learning loop

- Review feedback ingestion.
- Memory updates after merge/reject.
- Similar incident matching.
- Deploy-diff awareness.
- Incident recurrence detection.

## Phase 5: Multi-language support

- Python FastAPI/Django.
- Java Spring Boot.
- Go services.
- Ruby on Rails.
- PHP Laravel.

## Phase 6: Enterprise features

- SSO.
- RBAC.
- Team ownership mapping.
- Air-gapped deployment.
- Bring-your-own vector database.
- Approval workflows.
- Advanced policy controls.
